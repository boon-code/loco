Detailed Description

This is the main class that controlls the locating service.

This service is used to:

  • Locate the running device and respond by sms (CMD_LOCATE)
  • Parse response sms and show data (CMD_RECEIVE_RESULT_POSITION, CMD_RECEIVE_RESULT_CELLS)

int increaseNotifyID (  )  [private]

Helper method that increases the current notification id and returns it.

This method is used to simplify the handling of the id's used by NotificationManager.notify.

Note:
This method simply increments m_current_notify_id, counts up to MAX_NOTIFY_ID_COUNT - 1 and then starts over at 0. It's totally possible that an id is reused. However, this is not fatal, because this simply causes an old notification to be replaced by a new one and therefore limits the maximum number of notifications to MAX_NOTIFY_ID_COUNT.

Callback method (LocationListener), called on location update...

Added some additional tests because android os (2.2) calls this method with an outdated location but the current time even if the mobile data-connection is disabled. So if location updates happen and the location-provider isn't gps I consider it as false update and ignore it.

int onStartCommand ( Intent  intent,
int  flags,
int  startId 
)

Callback method (Service), the entry point that gets called if the service has been started with a specific intent (command).

Basically checks the intent for valid commands (EXTRA_KEY_CMD) and starts appropriate method that implement the commands. For now, 3 external commands are implemented:

  • CMD_LOCATE: Locates this device
  • CMD_RECEIVE_RESULT_POSITION: Shows position of tracked device.
  • CMD_RECEIVE_RESULT_CELLS: Shows cell information of tracked device.

There is a fourth command (CMD_WAKE) but it should only be used by this service (to wake the device on a periodic basis). CMD_ILLEGAL is used to indicate an invalid command, this is not a command, and shouldn't be used by external components.

Parameters:
intent The intent that started this service.
flags Additional flags...
startId Some id.
Returns:
A flag that determines whether the service should be restarted after it got killed by the os...
See also:
startLocating Method that registers a new request to locate the device
notifyPosition Method that adds a notification with the position of some other device.
notifyCell Method that adds a notification with some cell information that could be retrieved.

This command is used internally to wake the service periodically.

I thought that it's a good idea to wake the device to retrieve positional updates. Also, the location algorithm now works with a timeout (WAKE_UP_TIMEOUT_FAST, WAKE_UP_TIMEOUT_FAST). So it's essential to wake the device on a periodic basis and check m_best_location whether the location could be retrieved or only cell information can be sent to the requestor.
